/etc/passwd 

[사용자계정명][패스워드][UID][GID][코멘트][홈디렉토리][쉘]

① 사용자 계정
 - 사용자 계정을 나타내며 사용자 정보를 가지고 있는 여러 파일에서 
   사용자에 대한 정보를 찾는 기준이 된다. 사용자의 이름은 중복되지 않아야 한다.
 
② 패스워드
 - 패스워드가 암호화되어 /etc/shadow 파일에 저장되어 있음을 나타낸다.
 - 'x'로 표시되면 사용자의 필드가 암호화되어 /etc/shadow 파일에 저장되어 있음을 의미 한다.
 - * 표시 처리해주면 로그인이 불가능 (암호가 틀리다고 출력)

 ③ UID (User IDentification)
 - 사용자 개인을 숫자로 구별하는 고유 ID이며, root는 0번, 0~99번은 시스템 관리 계정을 위해
   예약되어 있으며,일반 사용자는 배포판에 따라 500번부터 시작한다.
 - 퍼미션을 변경할 경우에도 사용자 이름 대신에 UID를 이용하여도 시스템은 인식 한다.
 - 여러 사용자가 동일한 UID를 갖게 될 경우 각 사용자는 동일한 퍼미션을 갖게 된다.
 
 ④ GID (Group IDentification)
 - 각 사용자 계정은 고유한 그룹 ID를 가지고 있다. (관리자 그룹으로 0번이다.)
 - 사용자가 소속되어 있는 그룹을 나타내며, 0~99번은 시스템 계정을 위해 예약되어 있으며,
 - 사용자 그룹을 구분하기 위해 기본적으로 500번부터 지정 가능하다.
 - 그룹 번호와 이름은 /etc/group 파일에 저장된다.
 - 한사용자는 여러 그룹에 속 할 수 있다.

 ⑤ 사용자 정보 (Comment)
   (시스템 설정에 별다른 영향이 없다. 그래서 아무 이름으로 입력해줘도 된다.) 
 - 사용자 계정에 대한 주석문을 정의하며, 주로 전체 이름을 기록한다.
 - 반드시 기록해야 하는 것이 아니며, 시스템 설정에 별다른 영향을 주지 않는다.

 ⑥ 홈디렉토리
 - 사용자의 홈 디렉토리를 위치이다. (관리자이므로 홈 디렉토리가 /root 이다. 
   일반 사용자일 경우 /home 디렉토리 하위에 위치한다.)
 - 사용자가 로그인할 때 가장 처음에 위치하는 홈 디렉토리이다.
 - 반드시 절대 경로명으로 기록하며, 관리자(root)를 제외하고 
   대부분 /home 밑에 계정명으로 생성이 된다.
 
 ⑦ 로그인 쉘
 - 사용자의 쉘을 정의한다. (기본 설정은 bash 이다.)
 - 사용자가 로그인 시 사용할 쉘 파일의 위치를 정의 한다.
 - /etc/shells 파일에 정의되어 있으며, 기본 설정은 /bin/bash 이다.

[암호 변경 경고 일수]:[계정 사용 불가 날짜]:[계정 만료일]:[예약]

① 사용자 명
 - 사용자 계정을 나타낸다.
 - /etc/passwd와 /etc/group 파일과 함께 동일한 사용자의 이름이 존재하여야 한다.
 - 패스워드 필드에 관련하여 추가로 제공하는 역할을 한다.
 
 ② 패스워드
 - 암호화된 사용자의 패스워드가 저장된다. 
 - 사용자의 패스워드가 암호화되어 저장되어 있는 곳이다.
 - 처음 두 글자 $6 :SHA512(86 character) $1:MD5 $2 blowfish $5: SHA256
 - SALT값 운영체제가 생성 랜덤하게 생된 값과 사용자 암호를 CRIPT라는 함수를 이용하여
   암호화. 같은 패스워드라도 매번 다른 결과를 출력. 즉 랜덤

 ③ 최종 암호 변경일
 - 1970년 1월 1일부터 계산하여 마지막으로 패스워드를 바꾼 날짜까지의 값(단위 : 일)
 - 예) 15728 -> 1970년 1월 1일부터 15728일 경과
 - 1970년 1월 1일 기준으로 몇 일이 지났는지 그 일수로 표현이 된다.
 
 ④ 암호 변경 최소 일수
 - 패스워드를 변경하고자 하는 최소 일수가 기록
 - 최종 암호 변경일을 기준으로 최소 일수가 지난 후에만 패스워드 변경이 가능 하다.
 
 ⑤ 암호 변경 최대 일수 
 - 패스워드를 바꾸지 않고 최대한 사용할 수 있는 기간이다. (일)
 - 마지막으로 변경된 패스워드의 날짜를 기준으로 현재 사용하고 있는 패스워드를 
   변경하지 않고 사용할 수 있는 일 수 이다.

 ⑥ 암호 변경 경고 일수
 - 패스워드 최대 사용 기간에 가까워질 경우 사용 기한 며칠 전에 경고를 보낼 것인지 지정
 - 암호 변경 최대 일수가 얼마 남지 않았음을 경고하는 일수 이다.
 - 암호 변경 최대 일수가 경고 일수와 같거나 작을 경우 시스템에 로그인할 때마다 
   패스워드 변경 최대 일수까지 남은 일수를 메세지로 출력 한다.

- 예) 암호변경 최대 일수를 100으로 설정하고, 경고 일수를 10이라고 설정한 경우 
      90일 동안은 로그인 시 아무런 메시지가 없으나 91일부터 패스워드를 변경하라고 
	  경고 메세지를 보여준다.

 ⑦ 계정 사용 불가 날짜
 - 계정에 대한 사용 제한을 설정하고 며칠 후에 완전히 사용 정지할지 설정한다.
 - 설정된 날짜까지 로그인 하지 않을 시 사용자 계정은 Locking 된다.

 ⑧ 계정 만료일
 - 사용자가 시스템을 마지막으로 사용할 수 있는 날짜를 기록 한다. (계정 만기일)
 - 사용자 로그인할 때 가장 처음에 위치하는 홈 디렉토리이다.

 ⑨ 예약
 - 예약된 필드로 현재는 사용되지 않는다.
 - 사용자가 로그인 시 사용할 쉘 파일의 위치를 정의

/etc/group 
[그룹이름][패스워드][그룹의GID][그룹의구성원]

① 그룹 이름
 - 그룹 이름을 나타낸다.
② 패스워드
 - 그룹의 암호화된 패스워드 필드로 'x'로 표시된 경우 /etc/gshadow 파일에 실제 
   패스워드가 저장되어 있음을 의미 한다.
③ 그룹의 GID
 - 사용자를 UID로 구별하듯 그룹도 그룹 이름과 GID 두 가지로 각각 구별한다.
④ 그룹의 구성원
 - 각 사용자들은 콤마(.)로 구분하게 되며 한 그룹에 여러 사용자가 포함될 수 있다.
----------------------------------------------------------------------------------------- 
/etc/gshadow
[그룹명][그룹 패스워드][그룹관리자][그룹구성원]

① 그룹명
 - 그룹 이름을 나타낸다.
② 그룹 패스워드
- x -> 암호가 없다
- 빈칸으로 되어있을 경우 그룹 구성원만 그룹으로 로그인 할 수 있다.
 
- !로 표시 되어있을 경우 암호가 설정되지 않았으므로 아무도 newgrp 명령어를 
   사용하여 이 그룹에 가입할 수 없다.

- !!로 표시 되어있을 경우 !표시와 같지만, 한가지 차이점은 암호가 
   이전에 설정된 적이 없다는 것을 의미 한다.
 
- 암호가 설정된 경우, 그룹에 속하지 않는 사용자는 newgrp 명령어를 
  사용하여 그룹에 가입할 수 있다. 
   
③ 그룹 관리자
 - gpasswd 명령어를 이용하여 그룹 구성원을 추가하거나 삭제할 수 잇는 그룹 관리자 목록이다. 
   ( 각 관리자는 콤마(.)로 구분된다. )
   
④ 그룹 구성원
 - 관리자가 아닌 일반 그룹 구성원 목록이다. ( 각 구성원은 콤마(./,)로 구분된다. )
