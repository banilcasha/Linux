<파일 디스크립터 (FD)>
:리눅스의 일반 파일이나 장치파일을 포함한 모든 파일을 관리하는 방식을 의미

-표준입력장치 (키보드) fd/0 =stdin
-표준출력장치 (모니터) fd/1 =stdout
-표준에러장치 (모니터) fd/2 =stderr

<리다이렉션(Redirection)>
:표준 입력과 출력의 방향을 재지정하는 방법
명령어 입력        (키보드/표준입력)
명령어 결과 출력 (모니터/표준출력)

<출력 리다이렉션>
-모니터에 나온 모든 내용을 다 파일로 만들 수 있다
-단, 내용 덮어쓰기됨
[명령어] 1> [파일이름] : 파일 생성 및 덮어쓰기
  cat       1>  file_test (1>에서 1생략가능)
=cat        >   file_test]
# cat    [복사할파일] > [담을 파일] : 내용을 아예 다 담을 수 있다.
# ll > l_test : ll 명령어를 쳐서 나오는 출력이 l_test로 들어간다.

<Bash 쉘에서 파일 덮어쓰기 방지하는 설정>
#set -o noclobber		//덮어쓰기 방지 Enable
#set +o noclobber		//덮어쓰기 방지 Disable


<추가 리다이렉션>
-덮어쓰기가 아니고 이어서 써진다
[명령어] >> [파일명]
#ls >> ls_test
#date >> ls_test -> ls한 결과와 date의 결과가 추가된다


<오류 리다이렉션>
#ls errtest -> "ls: cannot access errtest: 그런 파일이나 디렉터리가 없습니다" 메세지 출력됨
#ls errtest 1> err1 
: 1>은 표준출력내용을 집어넣으라는 명령어이다.
그런데 표준출력이 없기 때문에 err1에는 아무것도 없음
#ls errtest 2> err1
: 2> 에러 메세지를 err1에 입력하라는 것이므로 에러메세지가 들어간다.


<오류메세지 출력하지 않게 하기>
/dev/null : 리눅스 특수장치 파일로서 파일의 내용을 지우거나 오류메세지를 해당파일로 리다이렉션하여 출력하지 않을 수 있다.
#cat ls_test 1> /dev/null :무반응
#cat ls_test 2> /dev/null :ls_test의 내용이 출력됨
:cat ls_test 의 값은 표준출력만 갖고있다.
그런데 1>로 보내면 표준출력의 내용을 다 보내라는 뜻이므로
화면에 출력되는 값이 없다.
그러나 2>는 에러출력을 보내라는 뜻이므로 보내지지않은 표준출력의 결과가 출력된 것이다.


<표준 입력 리다이렉션>
#tail -1 < test_re > test_123
: tail -1 test_re의 결과 값이 test_123에 저장됨
이 값은 < 을 생략해도 나오기 때문에 잘 쓰이지 않는다

<환경변수>

쉘 환경변수의 종류
1. 지역 환경변수 (쉘 변수)
2. 전역 환경변수 (환경 변수)

전역 환경변수의 종류
$PATH	 	: 실행파일을 찾을 경로
$HOME	 	: 현재 사용자의 홈 디렉토리
$LANG	 	: 쉘에서 사용하는 언어
$SHELL	 	: 로그인 시 사용하는 쉘 정보
$HISTSIZE 	: 히스토리 파일의 크기, 열 수
$HOSTNAME 	: 호스트 이름
$PS1		: 프롬프트

사용법
#echo [환경변수 명]

전역 환경변수 선언
# export [변수명]=값

지역 환경변수 선언
# [변수명]=값 **변수명과 값 사이에 공백이 있으면 안됨 
ex) # LINUX=123
      # echo $LINUX -> 123

초기화
# unset [변수명]



확인하는 법
# /bin/bash : 현재 쉘안에 다시 쉘을 만들겠다는 것
# echo [지역 환경변수] : 아무일도 없음
# echo [전역 환경변수] : 값 나옴
# exit 		: 쉘 밖으로 나오겠음
